Role & Context:
أنت مطور Full-Stack خبير. لدي تطبيق مبني بـ React/Next.js يحمل اسم "دوائي" (Dawai)، وقد واجهت بعض المشاكل البرمجية (Bugs) المتعلقة بالتوجيه (Routing) واستجابة الأزرار، بالإضافة إلى حاجتي لتوسيع منطق التطبيق (Business Logic). 

الرجاء كتابة وتصحيح الكود بناءً على المتطلبات التالية، مع التأكد من أن الكود نظيف، يعمل بدون أخطاء، ومقسّم إلى Components:

1. Fix Existing Bugs & State Management:
- أزرار التطبيق (مثل زر النقر على الدواء وزر تسجيل الدخول) لا تعمل. الرجاء التأكد من إضافة `onClick` handlers صحيحة.
- استخدم `React Router` أو `Next.js Navigation` لضمان الانتقال السليم بين الصفحات.

2. Authentication & Session Persistence (تسجيل الدخول الدائم):
- اجعل صفحة `Login/Signup` هي الصفحة الأولى (Landing Page) لأي مستخدم غير مسجل.
- عند نجاح تسجيل الدخول، قم بحفظ حالة المستخدم (User Session) في `localStorage` أو `Cookies` (مثال: { id, name, role, token }).
- في المرات القادمة، استخدم `useEffect` أو `Middleware` للتحقق من وجود الجلسة؛ إذا كان مسجلاً، قم بتوجيهه فوراً إلى داخل التطبيق دون المرور بصفحة تسجيل الدخول.

3. Role-Based Access Control (RBAC):
- في واجهة تسجيل الدخول/إنشاء الحساب، أضف خياراً (Toggle أو Tabs) لتحديد نوع الحساب: "مستخدم عادي" (Patient) أو "صيدلية" (Pharmacy).
- بناءً على الـ Role الذي تم اختياره:
  * إذا كان `role === 'patient'` يوجه إلى صفحة البحث عن الأدوية (الرئيسية).
  * إذا كان `role === 'pharmacy'` يوجه إلى مسار محمي جديد `/pharmacy-dashboard`.

4. Pharmacy Dashboard (لوحة تحكم الصيدلية):
- أنشئ لوحة تحكم خاصة لأصحاب الصيدليات تحتوي على واجهة لإدارة المخزون (CRUD Operations).
- الميزات المطلوبة: إضافة دواء جديد، تعديل السعر، تحديث الكمية المتاحة، وحذف دواء.

5. Prescription Validation Logic (رفع الوصفة الطبية):
- أضف خاصية للأدوية في قاعدة البيانات/الـ State تسمى `requiresPrescription: boolean`.
- عند محاولة المستخدم العادي طلب دواء يحمل هذه الخاصية (True)، أظهر نافذة منبثقة (Modal) تمنع إتمام الطلب وتطلب التالي:
  1. زر لرفع/التقاط صورة للوصفة الطبية (Image Upload Component).
  2. حقل إدخال لتاريخ الوصفة الطبية (Date Picker).
- أضف منطق تحقق (Validation): يجب ألا يتجاوز تاريخ الوصفة الطبية 6 أشهر من تاريخ اليوم. إذا كان أقدم من 6 أشهر، أظهر رسالة خطأ واضحة (مثال: "عذراً، الوصفة الطبية منتهية الصلاحية. يجب ألا يتجاوز تاريخها 6 أشهر").

Technical Execution:
- استخدم TypeScript لضمان الـ Type Safety.
- حافظ على تصميم الـ UI/UX السابق (Tailwind CSS & Framer Motion).
- قدم لي أكواد الـ Context API (أو Zustand) الخاصة بإدارة حالة المستخدم (Auth State).
